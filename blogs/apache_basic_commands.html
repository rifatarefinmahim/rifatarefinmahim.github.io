Basic Commands of Apache Server

01-Jan-2045

Apache HTTP Server (or simply Apache) is an open-source web server software developed by the Apache Software Foundation. It's widely used to host and serve websites and web applications on Linux systems.

Key Features

  • Open-source & Free
  • Modular Architecture (e.g., mod_ssl, mod_rewrite)
  • Cross-platform
  • .htaccess Support
  • Virtual Hosts support
  • Strong Logging and Security features

Apache on Linux – Common Commands

Install Apache
sudo apt update
sudo apt install apache2
apache2 --version //to check apache installed
Start/Stop/Restart Apache
sudo systemctl status apache2
sudo systemctl start apache2
sudo systemctl stop apache2
sudo systemctl restart apache2
Enable at Boot
sudo systemctl enable apache2
sudo systemctl is-enabled apache2 //shows whether Apache is enabled at boot

This command tells our Linux system to automatically start the Apache service at boot time.

Disable Apache from starting at boot
sudo systemctl disable apache2

This removes the autostart link, so Apache will not start automatically at boot. But it doesn't stop the service immediately — it just prevents it from running on startup.

Default Web Directory

Default web directory of apache webserver is /var/www/html. Place your index.html or PHP files here.

Configuration Files
  • Main Config - /etc/apache2/apache2.conf
  • Virtual Hosts - /etc/apache2/sites-available/000-default.conf
  • Ports Config - /etc/apache2/ports.conf
  • Enabled Modules - /etc/apache2/mods-enabled/
Main Config - /etc/apache2/apache2.conf
  • This is the primary configuration file for Apache.
  • It includes global settings that affect the whole server.
  • Contains directives like Timeout, KeepAlive, ServerRoot, and permission settings for directories.
  • Often includes other configuration files using Include or IncludeOptional.
Virtual Hosts - /etc/apache2/sites-available/000-default.conf
  • This is where we configure virtual hosts.
  • Virtual hosts allow Apache to serve multiple websites/domains from a single server.
  • This file defines the default website (usually for IP-based access or when no other host matches).
  • It includes directives like:
    <VirtualHost *:80>
        DocumentRoot /var/www/html
        ServerName example.com
    </VirtualHost>
  • We can create additional .conf files for other sites and enable them using
    sudo a2ensite yoursite.conf
    sudo systemctl reload apache2
Ports Config - /etc/apache2/ports.conf
  • This file tells Apache which ports to listen to.
  • Typically includes -
    Listen 80
    Listen 443
  • Sometimes includes additional logic to listen based on modules (e.g., SSL).
  • Works together with the Virtual Host definitions.
Enabled Modules - /etc/apache2/mods-enabled/
  • Actual module config files are stored in /etc/apache2/mods-available/.
  • To enable a module -
    sudo a2enmod rewrite
    sudo systemctl restart apache2
    This creates a symlink in mods-enabled/ pointing to the config in mods-available/.
  • Works together with the Virtual Host definitions.

Test Apache

After installing Apache, visit -

  • http://localhost
  • http://your_server_ip

You should see the "Apache2 Ubuntu Default Page".

Security Tips

  • Disable unnecessary modules
  • Use HTTPS (e.g., via Let's Encrypt)
  • Disable directory listing
  • Set proper file permissions and use .htaccess rules
